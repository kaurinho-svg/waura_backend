# Как добавить новый магазин (клиента) в бот

Этот бот построен на **мультитенантной архитектуре**. Это значит, что один и тот же сервер может обслуживать десятки и сотни разных Telegram-ботов для разных магазинов (клиентов) одновременно. Базы данных товаров и покупателей у каждого магазина изолированы.

Чтобы добавить нового клиента (новый магазин), писать код не нужно! Следуйте этому простому алгоритму:

## Шаг 1: Создание бота в Telegram
1. Клиент (или вы) заходит в [@BotFather](https://t.me/BotFather) в Telegram.
2. Пишет `/newbot`, придумывает имя (например, `Zara KZ`) и юзернейм (например, `zarakz_bot`).
3. Копирует **Bot Token** (выглядит как `123456789:ABCDefghijklmnopqrstuvwxyz`).

## Шаг 2: Получение Telegram ID владельца
Владельцу магазина нужно узнать свой цифровой `telegram_id` (чтобы бот признал его Главным Администратором).
1. Пусть клиент напишет боту [@userinfobot](https://t.me/userinfobot).
2. Бот пришлет его ID (например, `987654321`).

## Шаг 3: Добавление магазина в базу данных
Теперь нужно внести этого клиента в вашу базу данных Supabase.
1. Зайдите в панель управления **Supabase** -> `Table Editor` -> `bot_stores`.
2. Нажмите **Insert Row** (добавить запись).
3. Заполните обязательные поля:
   - `name`: Название магазина (например, "Zara KZ").
   - `telegram_id`: Вставьте ID владельца из Шага 2 (`987654321`). Это сделает его главным админом.
   - `bot_token`: Тот самый длинный токен от BotFather из Шага 1.
   - Остальные поля (`kaspi_phone`, `is_subscribed` и т.д.) можно оставить пустыми или заполнить значениями по умолчанию.

## Шаг 4: Перезапуск сервера (чтобы бот «проснулся»)
Поскольку сервер считывает список токенов только при запуске программы, нам нужно его перезагрузить.

1. Зайдите в вашу панель управления **Render** (или на ваш сервер).
2. Откройте сервис `telegram_bot_backend`.
3. Нажмите справа вверху **Manual Deploy** -> **Deploy latest commit** (или перезапустите сервис).

В этот момент сервер перезагрузится, скачает из таблицы `bot_stores` уже несколько токенов и запустит всех этих ботов параллельно.

### Как понять, что всё успешно?
В логах Render (вкладка Logs) вы увидите:
```
🤖 Starting bot for store: Waura Demo
🤖 Starting bot for store: Zara KZ
🚀 Started 2 shop bot(s) + scheduler
```

### Итог
Новый клиент платит вам деньги за подписку -> вы добавляете его токен и ID в табличку -> перезапускаете сервер -> у него начинает работать точно такой же бот со своей личной админкой (`/admin`) и своими уникальными товарами. Ни строчки кода писать не нужно!
